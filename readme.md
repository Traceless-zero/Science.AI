# Science.AI-demo 概念原型 README

## 一、项目核心定位与设计理念
**产品名称**：Science.AI
**一句话定义**：学术思想市场的 AI 增强型公共基础设施——让论证质量变得可见，让思想传承变得可追溯。
**核心定位**：在预印本与期刊之外，构建一个独立于发表权力的、以论证检验和公共辩论为驱动的第三层学术交流空间。

**三条核心准则**：
1. **身份平等，论证优先**：以论证质量为唯一评判尺度，不预设身份门槛
2. **AI守底线，人做判断**：AI仅校验论证形式规范，不评判观点对错，最终判断交还用户
3. **重沉淀，轻流量**：不做算法推荐，聚焦可引用、可追溯的公共知识沉淀

**生态定位**：SA 不是预印本服务器，不是期刊数据库，也不是传统学术论坛。它是**第三层学术交流空间**——以论证质量检验为核心，以 AI 辅助审查和结构化公共辩论为驱动，以可追溯、可引用的思想沉淀为目标的学术公共基础设施。

## 三、前端技术架构与规范
### 技术栈
- 视图层：纯原生 HTML + Tailwind CSS（CDN 引入，全局自定义配色）
- 交互层：原生 JavaScript，无前端框架依赖
- 图标：Font Awesome 6.4
- PDF渲染：PDF.js CDN（自适应容器尺寸、DPR高清抗模糊、分页控制）
- 数据：前端 Mock 数据驱动渲染，无后端依赖

### 核心架构模式
1. **单页动态页面加载（SPA）**：基于 `switchTab(tabName)` 动态请求 `pages/xxx.html` 页面片段，通过 `initPage(tabName)` 完成页面初始化，无需整页刷新
2. **自定义导航历史栈**：由于 SPA 不产生浏览器历史，实现了 `tabHistory` 数组 + `goBack()` 函数管理页面回退，支持跨模块跳转的完整返回链路
3. **数据驱动渲染**：所有内容统一从 `window.MOCK_DATA` 读取，JS 动态拼接 HTML 插入指定容器，视图与数据解耦
4. **防御性编码规范**：所有 DOM 获取、数据读取前均做非空校验，增加重试兜底逻辑，避免空值白屏报错
5. **统一组件体系**：卡片、按钮、标签、侧边色、hover动效、文本截断等全部复用统一 Tailwind 规范
6. **JS逻辑统一管理**：由于 SPA 通过 `innerHTML` 动态加载的页面片段中的 `<script>` 标签不会执行，所有核心交互逻辑（AI演示对话、辩论模式切换、质检面板等）均集中在 `common.js` 中管理

### 全局配色体系
- 主色：`primary: #1e40af`（全站统一蓝色系）
- 阵营色：正方蓝 `proSide: #2563eb`、反方红 `conSide: #dc2626`
- 状态色：通过绿（pass）、橙（flagged）、红（fail）三档标识
- 中性色：`dark: #111827`、灰阶体系沿用 Tailwind 默认

1. 先完成跳转校验，确保所有核心路径闭环，形成完整可演示的 MVP 原型
2. 固化当前代码规范与组件体系，新增功能沿用「数据驱动 + DOM非空校验 + 统一跳转函数」的统一模式
3. 正式工程化启动前，先明确技术栈升级方向：是否迁移至 Vue/React 框架、是否引入后端、是否对接真实 AI 能力，避免纯静态原型持续堆叠导致后续重构成本过高


## 补充部署说明
1. 项目必须通过Web服务运行，禁止直接双击本地HTML打开（file协议会拦截PDF.js本地静态PDF读取）
2. 全局PDF.js依赖CDN统一在根目录 `index.html` 头部引入，无需在 page.html 页面片段重复引入
3. 页面切换时序依托 `switchTab` + `initPage` 架构，切换 paper 页面自动初始化 PDF 渲染控件
4. SPA 架构下，所有核心交互逻辑（showDemoChat、switchDebateMode、toggleQCpanel、openPaperDetail 等）集中在 `common.js` 中管理，不依赖页面片段中的 `<script>` 标签
